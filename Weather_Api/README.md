# 🌦️ Proyecto: Weather API

## 📌 Descripción
El proyecto **Weather API** surge del reto #2 de [GeeksforGeeks - Backend Project Ideas](https://www.geeksforgeeks.org/blogs/backend-project-ideas/).  
Su objetivo es construir una **API RESTful** que permita consultar datos meteorológicos en tiempo real y gestionarlos como tareas.  

Actualmente el proyecto se encuentra en **fase de desarrollo**, por lo que algunas funcionalidades aún están en construcción.

---

## ⚙️ Tecnologías utilizadas
- **[Node.js](ca://s?q=Node.js):** Lógica del servidor.  
- **[Express.js](ca://s?q=Express.js):** Framework para construir la API.  
- **[OpenWeatherMap API](ca://s?q=OpenWeatherMap_API):** Fuente externa de datos meteorológicos.  
- **Procesamiento de [JSON](ca://s?q=Procesamiento_de_JSON):** Manejo de respuestas y datos.  
- **Diseño de [APIs RESTful](ca://s?q=APIs_RESTful):** Organización de endpoints y operaciones CRUD.  

---

## 🎯 Funcionalidades previstas
- **[Datos meteorológicos](ca://s?q=Datos_meteorologicos_API):**  
  - Consultar clima actual por nombre de ciudad.  
  - Retornar temperatura, humedad y condiciones.  
  - Implementar caché opcional para reducir llamadas a la API externa.  

- **[Gestión de tareas](ca://s?q=CRUD_de_tareas_Node.js):**  
  - Crear una tarea meteorológica asociada a una ciudad.  
  - Modificar detalles de la tarea (ciudad, descripción, estado).  
  - Marcar tareas como completadas o pendientes.  
  - Eliminar tareas por ID.

## 🎓 Objetivo de aprendizaje
Este proyecto busca reforzar:
- La **integración de APIs externas** en aplicaciones backend.
